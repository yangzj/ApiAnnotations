简介

现在，已经编写好第一个ApiGroup FirstAction，接下来我们就要开始见证成果的时刻了，看是否环境配置正确和程序编写是否规范。

目前项目启动我们是推荐使用Jetty，当然也是可以使用其他的服务器。还未安装Jetty 可查看jetty安装教程。

启动项目

启动项目，本项目使用的是Jetty启动

鼠标右键点击 Jetty Webapp 并选择 New 菜单项，新建 Jetty Webapp 启动项，如下图所示：

Project: 选择新建的项目（我的是demo）；WebApp Folder：src/main/webapp；Context Path：我定义的是demo(可选)；HTTP/HTTPs Ports: 设置为与项目、系统不冲突的端口；

配置信息填好之后，点击Debug，项目即可运行启动

启动异常处理

系统启动时，因为在启动列表中没有添加如下jar， 所以可能出现如下异常:java.lang.NoClassDefFoundError: cn/banbang/ApiAnnotations/web/gzip/GzipFilter; 具体如下图所示：

启动异常解决方案，打开起初Jetty配置的页面，勾选以下两个jar:ApiAnnotations-core.jar与ApiAnnotations-web.jar

重新启动项目，该异常将消失，说明项目已经正常启动。

开启浏览器看效果

打开浏览器在地址栏中输入: http://localhost:8082/demo/A10101，输出内容为：

{"error":0,"data":"hello world!"}

证明项目 框架搭建完成。同样，每次看到这个环节的时候，我就觉得很欣慰，一个项目搭建成功了。
