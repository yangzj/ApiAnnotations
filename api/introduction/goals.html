设计来源

当我们开发过程中，我们经常会想：我们为什么需要经常循环的写重复程序代码呢？为什么我们的程序越来越难维护呢？ 增删改查重复的编写，程序耦合度不断提高，项目API难以管理，项目组开发人员之间代码互相难以理解，众多图片视频资源难以管理， 我们不能够改善这些吗？

设计意图

我们致力于开发一套编写方便且便于维护开发框架，我们相信有明确意图并且简单明了的代码能够更好的提高开发速度以及维护。

设计思路

ApiAnnotations框架通过使用Java annotations，参照于AndroidAnnotations。开发者可用通过使用框架注解，并且根据框架规则编写javadoc， 然后ApiAnnotations在编译的时候根据注解和javadoc自动生成代码。同时平台提供一套项目API控制台； 通过云存储管理静态资源，以及提供一套静态资源管理控制台。

代码示例

/**
 * @author banbang
 * @description 用户角色类
 * @id 106
 */
@ApiGroup
public interface UserAction {
	/**
	 * 用户登入
	 * @id A10601
	 * @descriptions *手机号码/*密码/ip
	 * @filters open
	 * @return weixinOpenId;telephone
	 * @regep.telephone 1[0-9]{10}
	 * @regep.password (\S|\s){6,20}
	 */
	public User login(String telephone, String password, Ip ip);
}
